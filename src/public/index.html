Form Fields
Client Name

Any label that helps you identify the client internally. Can be the company name or a shorthand like "acme-ops".

SmartLead API Key
  1. Log in to SmartLead
  2. Click your profile avatar (top right) → Settings
  3. Go to the API Keys section
  4. Click Add New API Key, name it, and copy the key
Leave blank if this client doesn't run email campaigns.
HeyReach API Key
  1. Log in to HeyReach
  2. Go to Settings → Integrations → API Keys
  3. Click Generate New Key and copy it
Leave blank if this client doesn't run LinkedIn campaigns.
Slack Bot Token
  1. Go to api.slack.com/apps and open your SalesGlider Slack app
  2. Click OAuth & Permissions in the left sidebar
  3. Scroll to Bot Token Scopes and confirm these are added:
    • chat:write
    • chat:write.public
    • channels:history
    • im:write
  4. Scroll back up and copy the Bot User OAuth Token (starts with xoxb-)
Same token for all clients if they share the same Slack workspace. Different workspaces need different apps & tokens.
Slack Channel ID
  1. In Slack, create a dedicated channel for this client (e.g. #acme-replies)
  2. Invite the bot to the channel: type /invite @YourBotName in the channel
  3. Right-click the channel name in the sidebar → View channel details
  4. Scroll to the bottom — the Channel ID is listed there (starts with C)
Alternative: right-click channel → Copy link. The ID is the last segment of the URL.
Cal.com Event Type ID

An "event type" is a bookable meeting page on Cal.com — like a Calendly link. Each one has a numeric ID that goes here.

  1. Once the client's team and event type are set up (see the "Cal.com Setup" sections below), go to the team's Event Types
  2. Click the event type to edit it
  3. Look at the browser URL: cal.com/event-types/123456
  4. The number at the end is the Event Type ID — paste it here

Cal.com Setup (Per Client)
How Cal.com works — architecture overview

Cal.com ties into whatever calendar the client already uses (Google, Outlook, iCloud) and whatever video tool they use (Zoom, Google Meet, Teams). Once set up, every booking our system creates automatically:

  • Checks the client's real calendar for conflicts
  • Pushes the event onto their calendar
  • Generates a unique video call link
  • Sends confirmation emails & calendar invites to both parties
  • Handles reschedules, cancellations, and reminders

The structure:

  1. You own one Cal.com organization — "SalesGlider Growth"
  2. Each client gets their own sub-team under your org
  3. Each team has at least one event type (e.g. "30-Min Intro Call")
  4. The client's user is the team member — their calendar gets blocked, their name appears as the host
Step 1 — Create the client's sub-team
  1. Log in to cal.com
  2. Go to Teams in the sidebar → click Create Team
  3. Name it after the client (e.g. "Acme Corp")
  4. Skip billing — it rolls up to your org plan
Step 2 — Invite the client's user
  1. Inside the newly created team, click Members → Invite
  2. Invite the email of the person whose calendar meetings should land on — usually the founder, CEO, or whoever does discovery calls
  3. They get an email — they sign up for Cal.com (free) and join your team
If they already have a Cal.com account on the same email, the invite just adds them to your team — they don't need to re-sign-up.
Step 3 — Client connects their calendar

This is the critical step that makes availability checking + auto-booking work.

  1. The client logs into their Cal.com account
  2. Goes to Settings → Apps → Calendars
  3. Clicks Connect next to their calendar provider:
    • Google Calendar — OAuth flow, 30 seconds
    • Office 365 / Outlook — Microsoft OAuth
    • Apple iCloud — app-specific password
  4. After connecting, Cal.com asks which calendar to:
    • Check for conflicts (prevents double-booking)
    • Add events to (where new bookings appear)
  5. Select their primary work calendar for both
Test it: have them temporarily block 10am–11am tomorrow on their real calendar, then check their Cal.com booking page — that time slot should be unavailable.
Step 4 — Connect video conferencing (Zoom/Meet/Teams)

Same Settings → Apps section. Connect as many as the client uses:

  • Zoom — OAuth, 30 seconds. Cal.com auto-generates a unique Zoom link for every booking.
  • Google Meet — automatic if they connected Google Calendar. Every event gets a Meet link.
  • Microsoft Teams — OAuth if they're on Microsoft 365. Auto-generates Teams meetings.
  • Around / Whereby / Jitsi — also supported if the client uses them.

After connecting, go to Settings → Apps → set the default conferencing app to whichever one they prefer.

Step 5 — Create the event type
  1. Go to the client's team → Event Types → Create
  2. Fill in:
    • Title: e.g. "30-Minute Intro Call"
    • Duration: 30 min (or whatever they want)
    • Host: the client user you invited
    • Location: select Zoom / Google Meet / Teams — not "In-person" or "Link meeting"
    • Availability: leave as "default" — pulls from their connected calendar automatically
    • Email notifications: ON (Cal.com sends invites to both parties)
    • Reminders: optionally set 24h and 1h email reminders to the prospect
  3. Save
Step 6 — Grab the Event Type ID
  1. After saving the event type, stay on the edit page
  2. Look at the browser URL: cal.com/event-types/892341
  3. Copy just the number at the end
  4. Paste it into the Cal.com Event Type ID field in the client form
Step 7 — Test end-to-end

Before handing this off to the client, verify the full loop:

  1. Manually simulate a MEETING_PROPOSED reply through the system (or wait for a real one)
  2. In Slack, click Confirm & Book
  3. Verify:
    • The prospect receives a calendar invite email with a working video link
    • The event appears on the client's real Google/Outlook calendar automatically
    • The video link (Zoom/Meet/Teams) works when clicked
If anything fails, check Cal.com → Bookings to see the booking record and any errors.
After Creating the Client
Set up webhooks in SmartLead & HeyReach
  1. Click the client card in the dashboard to open its details
  2. Copy the SmartLead Webhook URL and paste it into your SmartLead campaign under Webhooks → reply-received trigger
  3. Copy the HeyReach Webhook URL and paste it into your HeyReach campaign webhook settings
  4. Send a test reply to verify the flow: webhook → classification → Slack approval message
